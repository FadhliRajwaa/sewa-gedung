📋 Detail Entitas Database

👨‍💼 ADMIN

  • id_admin - Primary Key
  • username - Unique Key
  • email - Unique Key
  • password - Hashed dengan bcrypt
  • nama_lengkap - Nama administrator
  • created_at - Timestamp pembuatan

👤 PENYEWA

  • id_penyewa - Primary Key
  • tipe_penyewa - individu/instansi
  • email - Unique Key
  • username - Unique Key
  • nama_instansi - Jika tipe instansi
  • nama_lengkap - Jika tipe individu
  • nik - Nomor Induk Kependudukan
  • email_terverifikasi - Status verifikasi

🏢 ACARA

  • id_acara - Primary Key
  • nama_acara - Jenis acara/gedung
  • kapasitas - Maksimal orang
  • harga - Harga per hari
  • lokasi - Lokasi gedung
  • status - tersedia/tidak tersedia
  • fasilitas - Daftar fasilitas

📅 PEMESANAN

  • id_pemesanan - Primary Key
  • id_penyewa - Foreign Key
  • id_acara - Foreign Key
  • tanggal_sewa - Tanggal mulai sewa
  • tanggal_selesai - Tanggal selesai
  • durasi - Durasi dalam hari
  • total - Total pembayaran
  • metode_pembayaran - QRIS/Transfer
  • tipe_pesanan - online/offline

💳 PEMBAYARAN

  • id_pembayaran - Primary Key
  • id_pemesanan - Foreign Key
  • bukti_pembayaran - File upload
  • tanggal_upload - Timestamp upload
  • status_pembayaran - Lunas/Belum Lunas

✉️ VERIFIKASI_EMAIL

  • id_verifikasi - Primary Key
  • id_penyewa - Foreign Key
  • token - Token verifikasi
  • created_at - Timestamp pembuatan
  • expires_at - Timestamp kedaluwarsa

🎯 Kardinalitas Relasi

🔗 PENYEWA ↔️ PEMESANAN

Kardinalitas: 1:N (One to Many)

Deskripsi: Satu penyewa dapat membuat banyak pemesanan

🔗 ACARA ↔️ PEMESANAN

Kardinalitas: 1:N (One to Many)

Deskripsi: Satu acara dapat dipesan berkali-kali (berbeda waktu)

🔗 PEMESANAN ↔️ PEMBAYARAN

Kardinalitas: 1:1 (One to One)

Deskripsi: Setiap pemesanan memiliki satu pembayaran

🔗 PENYEWA ↔️ VERIFIKASI_EMAIL

Kardinalitas: 1:N (One to Many)

Deskripsi: Satu penyewa dapat memiliki banyak token verifikasi

🔒 Fitur Keamanan Database

🔐 Password Security

  • Hashing dengan algoritma bcrypt
  • Salt otomatis untuk setiap password
  • Cost factor 10 untuk keamanan optimal

🗝️ Foreign Key Constraints

  • ON DELETE CASCADE - Hapus data terkait
  • ON UPDATE CASCADE - Update referensi otomatis
  • Referential integrity terjaga

✅ Data Validation

  • ENUM untuk nilai terbatas
  • NOT NULL untuk field wajib
  • UNIQUE untuk data unik (email, username)
  • Timestamp otomatis

📧 Email Verification

  • Token-based verification system
  • Expiration time untuk keamanan
  • Random token generation
